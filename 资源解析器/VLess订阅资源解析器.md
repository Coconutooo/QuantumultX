# VLess订阅资源解析器

## 📝 前言

在使用VLess订阅导入节点资源时出错，市面上现有的解析器只会报错，不能给我想要的结果，于是自己写了一个，用来把VLess转为VMess导入，从而实现导入节点资源

## 📋 功能简介
本工具用于将 VLess 格式的服务器订阅解析并转换为 Quantumult X 兼容的 VMess 格式配置文件。

## 🚀 快速开始

### 1. 环境要求
- Python 3.6+
- 依赖包：`requests`

### 2. 安装依赖
```bash
pip install requests
```

### 3. 使用方法
1. 修改 `subscription_url` 
2. 运行脚本：
```bash
python vless_parser.py
```

## 📁 输出文件

### VMess_Simple.conf
**最简配置** - 兼容性最佳，推荐优先使用
```
vmess=server:port, method=chacha20-poly1305, password=uuid, tag=节点名称
```

### VMess_Advanced.conf  
**完整配置** - 包含传输协议参数，功能更完整
```
vmess=server:port, method=chacha20-poly1305, password=uuid, obfs=wss, obfs-host=domain, obfs-uri=/path, tag=节点名称
```

## 🔧 配置说明

### 支持的传输协议
- **TCP** - 基础 TCP 连接
- **WebSocket (WS)** - `obfs=ws`
- **WebSocket over TLS (WSS)** - `obfs=wss` 
- **gRPC** - `obfs=grpc`
- **HTTP** - `obfs=http`

### 参数说明
| 参数        | 说明     | 示例                                   |
| ----------- | -------- | -------------------------------------- |
| `method`    | 加密方式 | `chacha20-poly1305`                    |
| `password`  | 用户UUID | `a8893c4c-8456-3410-b804-f46e4c9cf1d0` |
| `obfs`      | 传输协议 | `ws`, `wss`, `grpc`                    |
| `obfs-host` | 伪装域名 | `example.com`                          |
| `obfs-uri`  | 请求路径 | `/ws-path`                             |
| `tag`       | 节点名称 | `🇺🇸 美国节点`                          |

## 📱 Quantumult X 导入方法

### 配置文件导入

1. 打开 Quantumult X

2. 进入「设置」→「节点」

3. 点击「节点资源」

4. 选择右上角「银色链接图标」

5. 进入「资源-节点」页面,选择「资源路径」→「本地」

6. 选择生成的 `.conf` 文件

 ### 手动导入

1. 打开 Quantumult X
2. 进入「设置」→「节点」
3. 点击「节点资源」
4. 选择右上角「文件图标」
5. 将 `.conf` 文件中的配置信息粘贴进去（注意去掉换行）[参考样式](https://raw.githubusercontent.com/crossutility/Quantumult-X/master/server-complete.snippet "官方样式")
6. 点击保存，跳转到「资源-节点」页面，填写相关信息即可

## **🔔** 注意事项

1. **优先使用简单配置** - `VMess_Simple.conf` 兼容性最好
2. **节点命名** - 建议使用 icon + 地区 + 运营商格式
3. **安全提醒** - 不要公开分享包含 token 的订阅链接
4. **定期更新** - 建议定期更新订阅获取最新节点

## 🐛 常见问题

### Q: 导入后显示语法错误
A: 使用 `VMess_Simple.conf` 文件，该文件使用最基础的 VMess 格式

### Q: 节点连接失败  
A: 尝试使用 `VMess_Advanced.conf` 文件，包含完整的传输协议参数

### Q: 解析失败
A: 检查订阅链接是否有效，token 参数是否正确

### Q: 导入报错格式错误/Response invalid

A: Quantumult X 导入时会自动格式化，而输入规范要求规则之间没有换行,建议使用手动导入
